 # Auditoria Interna de 5S

 Site/diagnóstico 5S com páginas HTML, CSS e JS.

 ## Como publicar alterações

 1) Após editar arquivos:
```
git add .
git commit -m "sua mensagem"
git push
```

 2) Se preferir normalizar finais de linha no Windows (recomendado):
```
git config core.autocrlf true
```

 ## Organização sugerida
 - Manter apenas uma cópia das páginas (evitar duplicatas).
 - Pastas de backup e arquivos temporários não são versionados (.gitignore).

 ## Deploy (opcional)
 - GitHub Pages: Settings → Pages → Branch `main` (root).
 - Netlify: aponte o site para a raiz do repositório ou para uma pasta específica.
